# 经验总结技能

## 概述

此技能用于在**每次任务完成后**自动执行经验总结。无论任务成功还是失败，都会生成结构化的经验记录，帮助 Agent 和用户持续改进。

## 触发条件

- 任何任务执行完毕后（包括代码编写、调试、分析、搜索、文件操作等）
- 用户明确要求总结时
- 任务过程中发生错误或异常时
  
  ## 核心工作流
  
  ### 步骤 1：判断任务是否完成
  
  检查当前任务状态：
- 任务目标是否已达成？
- 用户是否确认完成？
- 是否输出了最终结果？
  如果任务未完成，**等待完成后再总结**。
  
  ### 步骤 2：生成经验记录
  
  按照以下格式生成总结：
  
  ## 📝 经验记录
- 时间：{当前时间}
- 任务类型：{代码/调试/分析/搜索/文件操作}
- 任务简述：{一句话描述做了什么}
  
  ### ✅ 完成情况
- 成功/失败/部分完成
- 如果失败，原因是什么
  
  ### 💡 关键发现
- {本次发现的重要信息或问题}
- {意外的收获或坑}
  
  ### 🔧 问题与解决
- 遇到问题：{是什么}
- 解决方法：{怎么解决的}
- 未解决问题：{如果有}
  
  ### 📌 学到的经验
- 下次类似任务应该：{做什么}
- 下次类似任务不应该：{做什么}
  
  ### 🚀 改进建议
- 对自己（Agent）的建议：{如何优化行为}
- 对用户的建议：{如果有必要}

### 步骤 3：写入经验文件

将生成的记录追加到 `.trae/skills/experience-learner/experience.log`

格式要求：

- 每条记录用 `---` 分隔

- 保持原始格式不变

- 不覆盖历史记录

### 步骤 4：简要告知用户

用一句话告知用户经验已记录，然后结束。

示例：

> ✅ 经验已记录，下次我会做得更好。

## 注意事项

1. 总结要**简洁**，不要超过 10 行

2. 没有发现问题时，仍要记录"一切正常，无特殊发现"

3. 同一任务多次执行时，每次都单独记录

4. 用户的纠正意见是**最宝贵的经验**，必须重点记录

## 示例

### 示例 1：成功的代码任务

用户：写一个读取 CSV 文件的函数

Agent 执行完成后自动总结：

markdown

## 📝 经验记录

- 时间：2026-04-28 14:30:00
- 任务类型：代码
- 任务简述：编写读取 CSV 文件的函数
  
  ### ✅ 完成情况
  
  成功
  
  ### 💡 关键发现
- 用户偏好使用 pandas 而非 csv 模块
- 函数需要处理文件不存在的异常
  
  ### 🔧 问题与解决
- 无特殊问题
  
  ### 📌 学到的经验
- 下次优先使用 pandas 处理表格数据
- 始终添加 try-except 处理文件读取
  
  ### 🚀 改进建议
- 下次可以先询问用户偏好哪个库

### 示例 2：失败的任务

用户：部署到生产环境（Agent 没有权限）

Agent 执行完成后自动总结：

markdown

## 📝 经验记录

- 时间：2026-04-28 15:00:00
- 任务类型：操作
- 任务简述：尝试部署到生产环境
  
  ### ✅ 完成情况
  
  失败
  
  ### 💡 关键发现
- 当前用户没有生产环境写权限
  
  ### 🔧 问题与解决
- 遇到问题：权限不足
- 解决方法：建议用户联系管理员开通权限，或使用 CI/CD
- 未解决问题：无
  
  ### 📌 学到的经验
- 下次先检查权限再尝试部署操作
- 敏感操作前应该主动询问用户确认
  
  ### 🚀 改进建议
- Agent 行为优化：部署前先执行 `ls` 或测试连接

## 经验文件的读取（可选）

如果用户要求查看历史经验：

1. 读取 `.trae/skills/experience-learner/experience.log`

2. 按时间倒序展示最近 5 条

3. 建议用户注意反复出现的错误类型
