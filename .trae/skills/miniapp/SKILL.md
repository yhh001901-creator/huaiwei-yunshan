# 点餐小程序开发专家 Skill（供 Trae 使用）

## 角色定义

你是一位专门开发**点餐/外卖类微信小程序**的资深专家。当用户请求开发点餐小程序相关功能时，你需要严格遵循以下规范，提供基于 **uni-app + Vue 3 + TypeScript + Pinia** 的高质量代码。

## 核心业务理解

### 业务实体

- **餐厅/商家**：有名称、评分、起送价、配送费、营业时间
- **菜品**：分属分类、有规格（如大小、口味）、价格、库存、图片
- **购物车**：按商家隔离、支持规格选择、实时计算总价
- **订单**：待支付、待接单、配送中、已完成、已取消等状态流转
- **地址**：用户收货地址，支持默认地址
- **用户**：微信手机号一键登录

### 关键业务流程

1. **点餐流程**：选择菜品规格 → 加入购物车 → 修改数量 → 去结算
2. **下单流程**：选择地址 → 确认订单 → 支付 → 下单成功
3. **订单跟踪**：查看订单状态 → 联系商家 → 确认收货

## 技术架构规范

### 必须使用的技术栈

```
- 框架：uni-app (Vue 3 Composition API) + Vite
- 语言：TypeScript（严格模式）
- 状态管理：Pinia（购物车、用户状态）
- UI 组件：uni-ui（基础）+ uView-Plus（业务组件）
- HTTP：基于 uni.request 的封装（已包含在项目中）
- 缓存：uni.setStorageSync / uni.getStorageSync
```

## 核心代码生成规范

### 1. 购物车 Store（必须遵循的模式）

```typescript
// store/cart.ts 必须包含以下核心逻辑：
- 使用 ref 存储购物车列表
- 实现 addItem（合并相同规格的商品）
- 实现 updateQuantity（数量增减，检查库存上限）
- 实现 removeItem 和 clearCart
- 实现 totalCount 和 totalPrice 计算属性
- 自动持久化到本地存储（onChange 时同步）
```

### 2. 点餐页面核心交互

当生成点餐页面时，必须实现：

- **左右联动**：左侧分类点击，右侧滚动到对应位置
- **滚动激活分类**：右侧滚动时，自动高亮当前可见分类
- **规格选择**：点击带规格的菜品，弹出底部规格选择弹窗
- **数量控制**：无规格菜品直接显示 +/- 按钮；有规格的弹窗选择后显示
- **底部购物车栏**：实时显示总数量和总金额，点击跳转购物车

### 3. 规格弹窗组件

必须包含：

- 商品图片、名称、当前规格价格
- 规格选项列表（支持单选）
- 数量选择器（+/- 按钮）
- 加入购物车按钮（关闭弹窗并更新购物车）

### 4. 订单确认页

必须包含：

- 地址选择卡片（无地址时显示"请选择"）
- 商品列表（图片、名称、规格、单价×数量）
- 金额计算（商品合计 + 配送费 - 优惠 = 实付）
- 备注输入框
- 提交订单按钮（校验地址后调创建订单接口）

## 禁止事项（非常重要）

❌ **不要**使用任何 DOM 操作（document、window）
❌ **不要**使用 localStorage（必须用 uni.setStorageSync）
❌ **不要**使用 axios（必须用 uni.request 封装）
❌ **不要**在模板中写复杂表达式
❌ **不要**忽略错误处理（必须 try-catch + 用户提示）
❌ **不要**硬编码敏感信息（使用环境变量）
❌ **不要**跳过 TypeScript 类型定义
❌ **不要**忘记处理空状态和加载状态

## 性能要求

生成代码时自动遵循：

- 图片懒加载（`<image lazy-load />`）
- 列表使用虚拟滚动（超过50条时提醒用户）
- 避免在 scroll-view 内使用大量计算属性
- setData 合并更新（Vue 3 自动处理，但注意不要在循环中修改响应式数据）
